what are categories of SQL Commands?

  • Data definition language (DDL) : These SQL commands are used to create ,modify and drop the structure of database objects like table , view ,procedure ,indexes etc . In this category we have CREATE ,ALTER ,DROP and TRUNCATE commands.

  • Data Manipulation Language(dml): These SQL commands are used to store ,modify and delete data from database tables. In this category we INSERT ,UPDATE and DELETE commands.

  • Transaction Control Language(TCL) : These sql commands are used to handle changes which affect the data in database. Basically we use these commands with in the transaction or to make a stable point, during changes in the database at which we can SAVEPOINT,ROLLBACK and COMMIT commands.

  • Data Control Language (DCL) : THese SQL commands are used to implemnt security database objects like table ,view ,stored procedure etc. In this category we have GRANT and REVOKE commands.

What are ER diagrams?

  • ER or entity relationship diagrams are used to model the logical view of the system from data prespective which consists of entity,entity type,entity set. An entity may be an object with physical existence -a particular person ,car ,house or employee. Or it can be an object with a conceptual existence - a company,a job or a university course.

  • Entity set means group of similar entities , for examaple class of students , here the entity is student.

  • Entity realationship diagrams show relationships between entities.

  • Types of relationships :

    • One to One Relationship : when each entity set can take part only once in the relation ship, the cardinality is one to one. Let us assume that a male can marry to one female and a female can marry to one male. So this relationship can be one to onel.

    • Many to one : when entities in other entity set can take part more than once in the relationship set. For example many students get enrolled in many courses.

    • Many to many : when entities in all sets can take part more than once in the relationship. For example many students can enroll in many courses.

what are keys in database?

  • key values in database can be attributes or set of attributes that can uniquely identify any row or record in any given table present in any database.

  • types of keys :

    • Primary key : primary key can be one coulmn or a set of columns which uniquely identify any record in a table. The primary key values for a particular record should be different and they can not be null.

    • Candidate key : candidate keys are those columns ,out of which one can be picked as a primary key , hence they act candidates for potential primary keys.

    • Super key : super keys are are basically superset of candidate keys , they are all the unique combinations or tuples of all the candidate keys.
